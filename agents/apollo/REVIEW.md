# Logicアプリ ビジュアルレビュープロセス

## 目的
コードレビューだけでなく、実際にアプリを目で見て使ってみることで、
ユーザー視点のバグ・UX問題を発見・修正する。

## レビューのタイミング
- **develop への機能追加・バグ修正 PR がマージされた後**
- **SIT デプロイ完了確認時**
- **本番デプロイ前の最終確認**

## レビュー手順

### 1. SIT 環境を開く
```
https://logic-sit.onrender.com
```
ブラウザツールで開き、ローカルストレージをクリアして新規ゲストとして開始。

### 2. チェックリスト（全画面を実際に操作する）

#### ホーム画面
- [ ] デイリーフェルミが JSON で返ってくる（HTML ではない）
- [ ] カテゴリカードの進捗 (done / total) が実データと一致
- [ ] 今週の記録が実際の学習日と一致（全チェックになっていない）
- [ ] プレースメントテスト未完了時に「Top XX%」が表示されない

#### デイリーフェルミ画面
- [ ] 問題文が表示される
- [ ] ヒントが問題に関連した具体的な内容（「問題が存在しません」ではない）
- [ ] 回答入力後に AI フィードバックが返ってくる

#### レッスン画面
- [ ] レッスン一覧が正しく表示される
- [ ] レッスンをクリックすると正しいコンテンツが開く

#### ロードマップ画面
- [ ] レッスン名とIDが一致している（フェルミ推定路が演繹法/帰納法を指している）
- [ ] 進捗カウンターが完了済みレッスンと一致

#### 統計画面
- [ ] 今週の記録が実際の学習日と一致
- [ ] 最近の活動に実際の完了レッスンが表示される（ダミーデータではない）
- [ ] プレースメントテスト未完了時はランキング非表示

#### プロフィール画面
- [ ] プランが実際の契約状態を表示（hardcodedではない）
- [ ] トライアル残日数が正確

#### ロールプレイ画面
- [ ] シナリオ一覧が表示される
- [ ] シナリオを選択して会話が始まる
- [ ] 採点画面が表示される（Scoring complete）

### 3. 発見した問題の記録
`agents/apollo/REVIEW_LOG.md` に日付・問題・修正内容を記録する。

### 4. 修正後の確認
- TypeScript ビルドエラーなし: `npx tsc --noEmit`
- 修正を develop にプッシュ → SIT 再確認

---

## 本番デプロイ前チェック
上記チェックリストを全て通過してから Keita さんに確認を取る。
承認後: `develop → main` マージ → Render 手動デプロイ
