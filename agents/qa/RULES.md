# QA Rules — 厳守事項

## 0. 原則
- **甘いテストは害悪。指摘ゼロまでテスト・修正を繰り返す**
- **見たことない画面は必ず触る**
- **報告書は嘘をつかない**（実際にやってないことを「やった」と書かない）

## 1. テスト前に必ず参照
1. `KNOWLEDGE.md` — Keita-san の過去フィードバック
2. `docs/DESIGN_V3.md` — デザイン仕様書
3. 該当チケットの受け入れ基準

## 2. 必須チェックリスト（全画面共通）

### 表示
- [ ] 背景は v3 ダーク（#082121 系）か
- [ ] 文字は読めるか（白文字 + 暗背景）
- [ ] カードが背景に埋もれていないか
- [ ] 画像が壊れていないか
- [ ] レイアウトが崩れていないか（PC/SP両方）

### インタラクション
- [ ] 全タップ可能要素を試す
- [ ] スクロールが滑らかか
- [ ] 入力フォーム（input/textarea）に入力できるか
- [ ] BottomNav が他要素と重なっていないか
- [ ] タップアニメーション（scale 0.98）が効くか

### 遷移
- [ ] 戻るボタンで戻れるか
- [ ] タブ切替で正しい画面に行くか
- [ ] CTAボタンが意図通りの画面に飛ぶか

### データ
- [ ] 実データが表示されるか（ハードコードでなく）
- [ ] 空状態の表示が適切か
- [ ] localStorage の状態を変えると反映されるか

## 3. テスト手順

### Step 1: ブラウザ準備
```
browser status → 起動確認
browser open → SIT URL
browser act resize → 390×844 (mobile)
```

### Step 2: localStorage クリア
```js
Object.keys(localStorage).forEach(k => localStorage.removeItem(k))
location.reload()
```

### Step 3: 全画面巡回
- ホーム → 全カードタップ
- レッスン → 全カテゴリタップ → 各レッスンタップ → Stories全進行 → 完了画面確認
- 記録 → 日/週/月切替 → タイムライン確認
- プロフィール → 全設定タップ

### Step 4: スクショ取得
- 各画面で必ずスクショ撮影
- 問題があれば箇所を明示

### Step 5: バグ報告
- 再現手順 / 期待値 / 実際 / スクショ を含める

## 4. Apollo に渡すレポート形式

```markdown

[コミットhash, 対象画面リスト]

- 失敗: X 件
- 警告: X 件

## 失敗詳細
### [画面名] - [問題サマリー]
- 再現手順: ...
- 期待値: ...
- 実際: ...
- スクショ: [path]

## 警告詳細（優先度低）
...

## 結論
- [ ] Keita-san に見せられるレベル
- [ ] 修正が必要（CTOに修正依頼を渡す）
```

## 5. ナレッジ更新義務
新しい指摘が出たら **必ず KNOWLEDGE.md に追記**してから次のテストへ。
