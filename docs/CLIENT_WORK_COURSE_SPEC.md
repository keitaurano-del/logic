## レッスン一覧

### Lesson 89：大きい数字の捉え方・概算力
**一言**: 「大きい数字を怖がらず、ざっくり扱える力」

| 項目 | 内容 |
|------|------|
| ねらい | 桁・単位・概算で数字を素早く把握し、ざっくり計算できるようにする |
| 扱う内容 | 10〜1兆の桁感覚、桁をずらして計算するコツ、単位変換、フェルミ推計の基本、AIの数字の妥当性チェック |
| ワーク例 | 100万×100万はいくらか、500万人×3000円で市場規模を出す、AIが出した数字が大きすぎないか確認する |
| 前提知識 | なし（入門からでも取り組める） |

**スライド構成（5ステップ）**:
1. `explain` — 桁感覚の基礎（10・100・1000・1万・10万・100万・1億・1兆の関係）
2. `explain` — 桁をずらして考えるコツ（100万×100万＝1兆の仕組み）
3. `quiz` — 100万×100万はいくらか（選択肢：1億／1兆／100兆／1京）→ 正解：1兆
4. `explain` — 市場規模の概算（500万人×3000円＝150億円の計算プロセス）
5. `quiz` — 「日本の年間コーヒー市場規模」をざっくり概算する（選択肢から最も近いものを選ぶ）

### Lesson 90：論点を定める
**一言**: 「解くべき問いを正しく設定する力」

| 項目 | 内容 |
|------|------|
| ねらい | クライアントの「言葉」の裏にある本当の課題を掴む |
| 扱う内容 | 論点と問いの違い、問いの設定ミスが招く失敗、So what？ / Why so？での深掘り、イシューを絞り込む考え方 |
| ワーク例 | 「売上が下がっている」という相談から本当の論点を探す、広すぎる問いを絞り込む |

**スライド構成（5ステップ）**:
1. `explain` — 「問いの設定ミス」が最大のロスになる理由
2. `explain` — So what？ / Why so？で論点を深掘りする
3. `quiz` — 「売上が下がっている」に対して最初に確認すべきことは何か
4. `explain` — イシューを絞り込む3つの問い（重要か・解けるか・今解くべきか）
5. `quiz` — 広すぎる論点を正しく絞り込む（選択肢から適切な論点を選ぶ）

### Lesson 91：ヒアリングの技術
**一言**: 「聞く力で、情報の質を上げる」

| 項目 | 内容 |
|------|------|
| ねらい | クライアントから必要な情報を引き出す質問設計ができるようになる |
| 扱う内容 | オープン質問とクローズド質問の使い分け、「5W1H」ではなく「論点ドリブン」のヒアリング、聞くべきことを事前に設計する、沈黙の使い方 |
| ワーク例 | NG質問とOK質問の比較、ヒアリングシートの骨格を作る |

**スライド構成（5ステップ）**:
1. `explain` — ヒアリングで最も多い失敗（聞きたいことを聞く vs 知るべきことを聞く）
2. `explain` — 論点ドリブンのヒアリング設計（「何を確認すれば仮説が検証できるか」から逆算）
3. `quiz` — NG質問とOK質問を見分ける（「何か困っていますか？」vs「どの場面で最も時間がかかっていますか？」）
4. `explain` — オープン質問・クローズド質問の使い分けと沈黙の技術
5. `quiz` — ヒアリングの場面で次に聞くべき質問を選ぶ

### Lesson 92：情報の読み解き方
**一言**: 「資料やデータから要点を掴む力」

| 項目 | 内容 |
|------|------|
| ねらい | 膨大な情報・資料から、必要な示唆を素早く取り出せるようになる |

| ワーク例 | 長い文章を「主張・根拠・具体例」に分解する、グラフから読み取れる示唆を言語化する |

**スライド構成（5ステップ）**:
1. `explain` — 「読む前に問いを持つ」ことで読解速度が上がる理由
2. `explain` — 主張・根拠・具体例に分解する読み方
3. `quiz` — 文章から「主張」にあたる部分を選ぶ
4. `explain` — AIの出力をそのまま受け取らないための点検（出典・論理の飛躍・数字の妥当性）
5. `quiz` — AIが出した情報の「おかしな点」を見つける

### Lesson 93：構造化思考
**一言**: 「考えをわかる形に並べ直す力」

| 項目 | 内容 |
|------|------|
| ねらい | 情報や考えを、流れ・粒度・抜け漏れ・関係性の観点から整理できるようになる |
| 扱う内容 | 話の流れのチェック、MECEなグルーピング、粒度をそろえる、因果関係と並列関係の見分け方、AIの出力を構造で検証する |
| ワーク例 | バラバラなメモを提案書の流れに整理する、AIが出した情報を「主張・根拠・具体例」に分ける、重複や飛躍を見つける |

**スライド構成（5ステップ）**:
1. `explain` — 「並べ方」が変わると伝わり方が変わる（構造化の重要性）
2. `explain` — 因果関係 vs 並列関係の見分け方、粒度をそろえる考え方
3. `quiz` — バラバラな4つの情報を正しくグルーピングする
4. `explain` — AIの出力を構造で点検する方法（抜け漏れ・重複・飛躍の検出）
5. `quiz` — 提案書の骨格として「おかしい順番」を見つける

### Lesson 94：伝え方・文章化
**一言**: 「考えを伝わる言葉に変える力」

| 項目 | 内容 |
|------|------|
| ねらい | 整理した内容を、相手に伝わる文章・表現に落とし込めるようになる |
| 扱う内容 | 結論から伝える、抽象と具体の使い分け、読み手に合わせた言い換え、長い文を短くする、AIの文章を自分の言葉に整える |
| ワーク例 | 箇条書きメモを提案書向けの文章に直す、同じ内容を「社内向け」「顧客向け」で書き分ける、AIが作った文をより自然な表現に修正する |

**スライド構成（5ステップ）**:
1. `explain` — 結論ファーストで伝える理由（PREP法の基本）
2. `explain` — 抽象・具体のバランス、読み手に合わせた言い換えの技術
3. `quiz` — 同じ内容の文章、より伝わるのはどちらか（2択）
4. `explain` — AIの文章をそのまま使わない理由と整え方
5. `quiz` — AIが作った文章を「より伝わる形」に修正する選択肢を選ぶ

3. `quiz` — 3つの事実を正しい順番に並べる
4. `explain` — 「So what?」でスライドをつなぐ技術
5. `quiz` — NG構成の「おかしい点」を見つける

### Lesson 97：フィードバックの受け方・活かし方
**一言**: 「批判を力に変える技術」

| 項目 | 内容 |

3. `quiz` — 「これじゃない」と言われたとき、次に聞くべき質問を選ぶ
4. `explain` — フィードバックを改善サイクルに落とし込む方法
5. `quiz` — フィードバックへの返し方として適切なものを選ぶ
