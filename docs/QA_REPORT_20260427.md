## 🔴 失敗詳細（修正必須）

### BUG-001: オンボーディング3 / プラン選択 — ベータキャンペーンカード文字が読めない
- **再現手順**:
  1. `localStorage.clear()` → `?qa=fresh` で SIT 開く
  2. オンボ Step1「はじめる」→ Step2「無料トライアルを開始」
  3. プラン選択画面で「ベータキャンペーン」が **デフォルト選択** された状態を見る
- **期待値**: プラン名「ベータキャンペーン」が読める
- **実際**: カード背景 `rgb(255, 251, 240)` ＝ 薄クリーム色 / タイトル文字色 `rgb(242, 247, 246)` ＝ ほぼ白文字 → **白に近い文字 on 白に近い背景** で完全に読めない
- **CSS診断（実測値）**:
  ```js
  // 選択中ベータキャンペーンカード
  cardBg: 'rgb(255, 251, 240)',   // クリーム
  titleColor: 'rgb(242, 247, 246)', // ほぼ白
  ```
- **修正方針案**:
  - 選択時の文字色を `var(--text-primary-dark)` (#0F1523 等) に固定
  - または選択時の背景を mint 系に変える（v3 ブランドと統一）
- **スクショ**: `docs/qa_screenshots_20260427/03_onboarding_step3.png`

### BUG-002: ロールプレイ選択画面 — 上半分が真っ白
- **再現手順**:
  1. ホーム画面の「ロールプレイ」大カードをタップ
- **期待値**: v3 ダーク背景（#082121）でカテゴリ＋シナリオ一覧が表示
- **実際**: 上半分が真っ白で何も表示されず、下半分にカードが寄ってる
- **原因推定**: SafeArea コンテナの背景色が未指定 / ヘッダー領域の `backgroundColor` が body 既定の `#fff` のまま
- **修正方針**: `RoleplaySelectScreen.tsx` のルート要素に `backgroundColor: 'var(--bg-base)'` 等を明示。または `body.mode-dark` の overrides に該当 selector 追加
- **スクショ**: `docs/qa_screenshots_20260427/46_roleplay_select.png`
