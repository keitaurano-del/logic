## 投稿案② — 言語化が全て

```
プログラミング経験ゼロでAI開発をやってみて、一番感じたこと。

コードが書けるかどうかは関係ない。「何を作りたいか」を言語化できるかどうかが全て。

UIのモックアップを何度も作り直した。v2、v3、v4と。毎回「これじゃない」と言って、「何がどう違うのか」をAIに伝え直す作業だった。

「統一感がない」では伝わらない。「青を1色に絞って、アイコンは塗りつぶし型にして、ネイビーの背景は廃止」まで言語化して初めてAIが動ける。

コンサルで要件定義や課題の構造化をやってきた経験が、そのまま効いている。
```

## 投稿案③ — 組織設計をAIに適用した話

```
Genspark Clawの中にCEO、CTO、CPO、CMO、COOの5つのAIエージェントを作った。

それぞれにルール、ナレッジ、承認フローを定義した。本番デプロイは承認必須。SITへのデプロイは自由。課金操作は必ず人間の確認を通す。

コンサルで組織設計をやっていた経験がそのまま使えた。

権限委譲の粒度、判断基準の明文化、レポートライン。AIに仕事を任せるのと、人に仕事を任せるのは構造が同じだった。

プログラミングの知識より、マネジメントの知識のほうが役に立っている。
```

## 投稿案④ — バグ10個を3時間で直した朝

```
ある朝、開発中のアプリのテスト環境を一通りチェックした。バグが10個出てきた。

APIが問題文ではなくHTMLを返す。画面間のIDがずれている。テスト未完了なのに「Top 50%」と表示される。ダミーデータが本番に残っている。

全部AIに伝えて修正させた。3時間で10件。

ただし、「何がおかしいか」を見つけたのは自分。AIは修正は速いが、異常の発見は苦手。

プログラミングの知識がなくても「これはおかしい」はわかる。ユーザー目線でアプリを触って違和感を言語化する。それがコードを書けない人間の仕事。
```

## 投稿案⑤ — AIにできないこと

```
AI駆動開発でも、AIでは解決できないことがある。

- Stripeの本番審査。コンプライアンス書類を人間が出すしかない
- Google Play Consoleの設定。手動で画面を操作するしかない
- AndroidのネイティブビルドにはMacが必要。クラウドVMではできない
- 外部サービスの契約手続き

コードの自動化と、開発の全自動化は違う。外部サービスとの接続部分には必ず人間の手作業が残る。

ここを理解しておかないと「AIに全部やらせよう」で詰む。
```

## 投稿案⑥ — コスト

```
プログラミング経験ゼロの人間がAIでアプリを作った場合のコスト。

Genspark Claw（プラン+追加クレジット）: 約65,000円
Claude Code: 約15,000円
Supabase: 無料枠
Render: 無料枠

合計で約8万円。正直、安くはない。


ただしエンジニアを1人雇うと月40〜80万円。外注なら100万円超もある。それと比べれば桁が違う。

開発が落ち着けば月2〜3万円まで下がる見込み。初月は試行錯誤が多かった。

コンサルの日当1日分にも満たない金額で、プロダクトが1本形になる。

エンジニアを雇わなくていいという話ではない。スケールするなら必要になる。でも「形にして検証する」段階ではこれで十分だった。
```

## 投稿案⑦ — 人間の仕事

```
AI駆動開発を数日やって、自分の役割がはっきりした。

自分がやること:
- 何を作るか決める
- 出来上がったものを触って判断する
- 「何がどう違うか」を言語化する
- 優先順位をつける

AIがやること:
- コードを書く
- デプロイする
- テストする
- ドキュメントを作る
- 修正する

プログラミング経験がなくても、上の4つはできる。むしろビジネスの現場で日常的にやっていること。

コンサルとして「課題を構造化して、優先順位をつけて、人に伝える」という仕事をしてきた人間にとって、AI駆動開発は自然な延長線上にあった。
```

## 投稿案⑧ — ツールスタック解説（スレッド用）

```
プログラミング経験ゼロでアプリ開発するときのツールスタック。

自分の環境を書いておく。

Genspark Claw — 専用VMが常時稼働。Ubuntu上にNode.js、Python、各種CLIがプリインストール済み。ここが開発の拠点。

Claude Code — Claw VM上で動くAIコーディングエージェント。ファイル探索、設計、実装、テストを一気通貫でやる。CLAUDE.mdにルールを書いておけば従う。

OpenClaw — AIエージェントの管理基盤。cronで定期タスクを回したり、Slack通知を飛ばしたり。

Supabase — データベースと認証。PostgreSQL + Google OAuth。

Render — ホスティング。developブランチをpushすればテスト環境に自動デプロイ。

Stripe — 決済。テスト環境で検証してから本番に切り替える。

全部クラウドサービスで、ローカルにIDEもエディタも入れていない。ブラウザだけで完結する。
```
